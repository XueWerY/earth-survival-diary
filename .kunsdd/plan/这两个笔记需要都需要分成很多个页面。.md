## Steps

### Step 1: 编写迁移脚本 `scripts/migrate-docs-to-notes.cjs`

**1.1 Markdown 解析器**：

- 按 `##`  和 `###`  标题行拆分内容
- 收集每个标题下的所有后续内容（直到下一个同级或更高级标题）
- 将 Markdown 内容按行转为 `<p>` 标签 HTML

**1.2 定位数据目录**：

- 从环境变量获取 `%APPDATA%\earth-survival-diary\data\`
- 遍历 `data/users/` 下所有 `.json` 文件获取用户列表

**1.3 为每个用户创建笔记**：

- 确保 `notes/categories.json` 存在（不存在则用默认分类初始化）
- 检查是否已有同名笔记（幂等，title 判断）
- 构建 Note 对象并追加到 `notes/notes.json`
- 内容格式：JSON 页面格式
  ```json
  { "pages": [
    { "id": "p_xxx", "title": "封面", "level": 1, "type": "cover", "content": "" },
    { "id": "p_xxx", "title": "一、项目概述", "level": 1, "content": "<p>...</p>" },
    { "id": "p_xxx", "title": "致谢", "level": 1, "type": "thanks", "content": "" }
  ] }
  ```
- 使用指南中 L2 页面设置 `parentId` 指向"功能特性"父页的 id

**1.4 删除源文件**：`项目规范.md` 和 `README.md`

### Step 2: 运行脚本

```bash
node scripts/migrate-docs-to-notes.cjs
```

`package.json` 中 `version` → `2026.7.8-4`
