地球 Online 生存日记 — 项目开发规范文档

01一、项目概述

地球 Online 生存日记是一款跨平台生活管理应用，将地球 Online 的游戏化概念融入日常时间管理，帮助玩家（用户）记录足迹、管理任务、专注计时、规划课程，并在 3D 地球可视化中呈现生存轨迹。

核心定位

  • 跨平台支持：Windows 桌面端、Android、HarmonyOS、浏览器
  • 游戏化体验：以"地球 Online 玩家"的身份管理生活
  • 3D 可视化：Three.js 渲染地球、月球、太阳及流星粒子
  • 本地优先：数据存储在用户本地，支持多用户隔离

功能模块

应用包含足迹记录、笔记、专注计时、清单、倒数日、课程表、统计、工具箱、插件系统等功能模块，覆盖日常生活管理的各个方面。

02二、开发环境配置

基础环境

  • Node.js 18+（推荐 20 LTS）
  • pnpm（包管理器，npm install -g pnpm 安装）
  • TypeScript 5.8+（随项目依赖安装）
  • Android Studio（Android 构建用，需配置 JDK 17+ 与 Android SDK）
  • DevEco Studio（HarmonyOS 构建用）

安装依赖

pnpm install

Electron 端服务依赖（express、cors）已包含在主 package.json 中，无需单独安装。生产打包时，electron/prod-server/package.json 用于将服务依赖复制到 electron/node_modules。

浏览器端开发调试

浏览器端开发是最快速的调试方式，适合日常 UI 与逻辑开发。数据通过 Capacitor Filesystem 的 Web 兜底实现存储到浏览器 IndexedDB 中，无需启动后端服务。

  1. 启动 Vite 开发服务器：
    npx vite
  2. 浏览器访问终端输出的地址（默认 http://localhost:5173）。
  3. 注册账号并登录，数据将存储在浏览器 IndexedDB 中（清除浏览器数据会丢失）。

调试技巧：

  • 使用 Chrome DevTools（F12）断点调试、查看 Network 与 Application（IndexedDB）面板。
  • 安装 Vue DevTools 浏览器插件，可查看组件树、Pinia 状态与路由。
  • 代码修改后 Vite 会自动热更新（HMR），无需手动刷新。
  • 浏览器端无法调试 Electron 主进程（IPC、窗口管理、系统托盘等），需在 Electron 环境中验证。

Electron 端开发调试

Electron 端通过内置 Express 服务器（electron/prod-server.cjs）提供 API 服务，数据存储在 app.getPath('userData')/data/ 目录下。

  1. 构建前端资源：
    pnpm build
  2. 启动 Electron 应用：
    npx electron electron/main.cjs

调试技巧：

  • Electron 主进程日志输出到终端与日志文件（userData/logs/app-YYYY-MM-DD.log）。
  • 渲染进程可通过 mainWindow.webContents.openDevTools() 打开 DevTools（生产环境已禁用，开发时需临时开启）。
  • 如需修改主进程代码，重启 Electron 即可生效。

HarmonyOS 端开发调试

HarmonyOS 端通过 scripts/harmony-sync.cjs 将构建产物同步到鸿蒙工程目录，由 DevEco Studio 编译运行。数据通过 window.harmonyAPI 原生接口存储。

  1. 构建并同步代码：
    pnpm harmony:build
  2. 用 DevEco Studio 打开鸿蒙工程目录，编译并运行到模拟器或真机。
  3. 仅同步代码不重新构建：pnpm harmony:sync

Android 端开发调试

Android 端基于 Capacitor，数据通过 Capacitor Filesystem 存储到应用沙箱目录。

  1. 构建前端资源：pnpm build
  2. 用 Android Studio 打开 android/ 目录。
  3. 同步 Capacitor 配置后编译运行：npx cap sync android

注意事项

  • 浏览器端与 Electron 端数据存储相互隔离，不互通。
  • vite.config.ts 中的 base: './' 配置确保打包后资源路径正确，请勿修改。
  • virtual:version 虚拟模块在构建时从 package.json 注入版本号，运行时通过 import 获取。
  • HarmonyOS 构建产物为 IIFE 格式，scripts/harmony-sync.cjs 会处理 import.meta.url 与 __vitePreload 兼容问题。

04四、技术架构

前端框架

  • Vue 3 + Composition API + TypeScript
  • Pinia 状态管理
  • Vue Router 路由
  • Element Plus UI 组件库

3D 可视化

  • Three.js 渲染地球、月球、太阳
  • NASA 真实卫星图片作为纹理
  • Canvas 流星粒子动画（z-index 低于 Three.js 元素）
  • MeshPhongMaterial 须包含 fallback 颜色 (0x2a6f97) 防止纹理加载失败黑屏

跨平台

  • Electron：Windows 桌面端，内置 Express HTTP 服务器
  • Capacitor：Android 平台
  • HarmonyOS：ArkWeb 引擎，需特殊处理 esbuild IIFE 打包

编辑器与图表

  • Vditor：笔记编辑器
  • ECharts：统计图表
  • lunar-javascript：农历日期计算

05五、编码规范

基本原则

  • 用最少的代码解决问题，不编写多余内容
  • 仅改动必要内容，不擅自优化周边代码
  • 遵循原有代码风格（缩进、命名、导入方式）
  • 不开发需求以外的额外功能
  • 单次使用的代码不做抽象封装

修改规则

  • 只清理自身修改产生的冗余
  • 不擅自删除原本就存在的无用代码
  • 删除本次修改后不再使用的导入语句、变量、函数
  • 不对运行正常的代码进行重构
  • 不对不可能出现的场景编写异常处理逻辑

验证原则

将任务转化为可验证的目标，完成后进行验证。对于多步骤任务，列出执行计划。

06六、UI/UX 规范

视觉风格

  • 深色星空背景主题，渐变色系（#0f0c29 → #302b63 → #24243e）
  • 主色调：#667eea（蓝紫渐变）
  • 强调色：#fbbf24（琥珀色，用于置顶/重要标识）
  • 文字颜色使用 rgba 白色透明度梯度

布局规范

  • 模块间使用面包屑地址栏导航，支持下拉快速切换
  • 卡片网格布局，响应式列数（最多 6 列）
  • 支持分屏显示（左右双面板）

交互规范

  • 卡片右上角 hover 显示操作按钮（编辑/删除）
  • 下拉菜单使用固定定位 + backdrop-filter 模糊背景
  • 确认操作使用 ConfirmDialog 组件
  • 消息提示使用 ElMessage

3D 场景层级

Canvas 元素（流星画布）的 z-index 必须低于 Three.js 元素（地球、月球、太阳），避免遮挡。

笔记幻灯片内容区

  • 幻灯片内容区（编辑/预览/放映）去除深色背景，直接融入应用星空主题，文字使用浅色
  • 通过工具栏"插入 HTML 代码"按钮插入代码块，编辑/预览/放映态均以语法高亮形式显示
  • 代码块（原生 pre）自动语法高亮着色和格式化缩进，使用 src/utils/codeBlock.ts 的 highlightCodeBlocks 和 serializeCodeBlocks 函数；编辑态保存前需调用 serializeCodeBlocks 将高亮标记还原为纯文本
  • 主工具栏在未选中文本时禁用字体、字号、加粗、斜体、下划线、删除线和字体颜色按钮，通过 hasSelection 响应式变量控制
  • 下划线按钮为单按钮（显示带下划线的"U"预览，点击仅展开下拉菜单不直接应用），下拉菜单提供 8 种线条样式、"无"和下划线颜色设置，菜单项只显示线条样式预览和名称；"下划线颜色"弹出的 ColorPickerPanel 通过 Teleport 浮动到 body 避免被编辑器容器 overflow:hidden 裁剪，并按视口动态计算 max-height 出现滚动条
  • 复制样式和应用样式按钮位于主工具栏，复制的样式以结构化对象保存到 data/用户ID/notes/style.json（字段含 fontFamily、fontSize、bold、italic、underline、strikeThrough、color、list、align、blockquote），应用时按这些状态逐项执行 execCommand；样式仅在当前编辑会话内有效，NoteEditor onBeforeUnmount 时调用 deleteData('notes','style') 清空持久化与内存
  • 粘贴内容时仅提供"保留源格式"和"只保留文本"两个选项；"只保留文本"以 font-family: 'Times New Roman', '宋体', SimSun; font-size: 10.5pt（五号）插入，英文走 Times New Roman、中文回落宋体
  • 致谢页通过右键侧边导航栏的致谢页导航项，在幻灯片菜单中选择"修改致谢内容"弹窗编辑

07七、组件使用规范

组件组织

  • 组件按功能模块组织在 src/components/ 下
  • 通用组件放在 common/ 下（nav、card、overlay、picker）
  • 每个模块的页面组件以 Page 结尾命名（如 NotesPage.vue）

常用组件

  • MainNav：全局底部导航栏
  • ReminderCard：提醒卡片（每 5 秒弹出一张）
  • ConfirmDialog：确认弹窗（v-model:visible 控制）
  • ColorPickerPanel：颜色选择面板（位于 picker/ 目录），自带 max-height: 90vh + overflow-y: auto；在编辑器内作为"下划线颜色"二级弹出时需 Teleport 到 body 并按视口动态计算 max-height，避免被祖先 overflow:hidden 裁剪
  • GuideOverlay：新手引导遮罩层

选择器组件

  • DateScrollPicker / InlineLunarDatePicker：日期选择
  • TimePickerPopover / ReminderTimePicker：时间选择
  • PeriodCountPicker：周期计数选择

08八、版本控制策略

版本号格式

版本号格式为 YYYY.M.DD-X，其中：

  • YYYY：年份（4 位）
  • M：月份（不补零，如 5 月写 5 不写 05）
  • DD：日期（不补零）
  • X：当日第 X 次发布（从 1 开始递增）

示例：2026.7.8-1 表示 2026 年 7 月 8 日第 1 次发布。

CHANGELOG 规范

  • 更新日志添加在 CHANGELOG.md 顶部（标题行下方）
  • 格式：### v<版本号> (<日期>)
  • 描述使用非程序员也能看懂的语言

发布流程

  1. 在 package.json 中更新 version 字段
  2. 在使用指南 HTML 中反映本次修改内容
  3. 在 CHANGELOG.md 顶部添加更新日志条目
  4. 如涉及基础性约定，同步更新项目规范 HTML
  5. 如涉及插件更新，更新插件版本号

09九、构建与部署规范

Windows 构建

pnpm electron:build:win

该命令依次执行：清理 release → vite build → 安装 Electron 依赖 → 生成图标 → electron-builder 打包 → 修复 latest.yml → 清理。

HarmonyOS 构建

pnpm harmony:build

先执行 vite build，再通过 harmony-sync.cjs 同步到鸿蒙工程。需注意：

  • esbuild IIFE 中的 import.meta.url 需后处理替换
  • Vite __vitePreload 机制需跳过 JS 预加载，仅保留 CSS

Android 构建

通过 Android Studio 打开 android/ 目录，执行 Gradle 构建。

发布

pnpm publish-release

发布到 GitHub generic provider，通过 electron-updater 实现自动更新。

10十、插件系统

插件结构

插件位于 src/plugins/ 目录下，每个插件包含：

  • index.ts：插件入口
  • plugin.json：插件元数据（名称、版本、描述）
  • tools/：插件工具目录，每个工具有独立的 Vue 组件

内置插件

  • file-manager：文件管理器（含 HTTP 服务器）
  • random-number：随机数生成器
  • reminder-viewer：提醒查看器

插件加载

通过 pluginLoader.ts 动态加载插件，工具箱页面展示已安装的插件列表。

11十一、数据管理

存储架构

  • Electron 端：通过 Express HTTP 服务器 + 本地文件系统（YAML 后端）
  • Capacitor 端（Android/HarmonyOS）：本地 localStorage 存储
  • 数据按用户 ID 隔离，路径为 data/<用户ID>/<类型>/<键>

系统状态

SystemState 存储在 system/state 中，包含各模块的状态字段。defaultsInitialized 标记是否已初始化默认数据。

项目文档

项目规范和使用指南已迁移为 HTML 文件：项目规范.html 位于仓库根目录，作为项目开发文档；使用指南.html 位于 public/ 目录随应用打包，应用内"我的"页面提供入口按钮在应用内查看。

关键数据路径

  • 窗口分辨率设置：data/<用户ID>/settings/settings.json
  • 清单收藏和快速访问：data/<用户ID>/list/ 目录
  • 笔记数据：data/<用户ID>/notes/notes.json
  • 笔记分类：data/<用户ID>/notes/categories.json
  • 笔记样式：data/<用户ID>/notes/style.json（复制样式保存，结构化对象含 fontFamily/fontSize/bold/italic/underline/strikeThrough/color/list/align/blockquote 字段；仅当前编辑会话内有效，离开 NoteEditor 时由 deleteData 清空）

缓存机制

storageService 使用内存缓存（Map）减少重复请求，clearCache() 可清除缓存。
