项目概述

了解项目定位与核心功能模块

本地优先

02

开发环境配置

搭建本地开发环境到各平台调试

基础环境

📦

Node.js 18+

推荐 20 LTS

⚡

pnpm

npm install -g pnpm

📘

TypeScript 5.8+

随项目依赖安装

安装依赖 
pnpm install

Electron 端依赖在 electron/ 目录通过 electron/package.json 管理，由构建脚本 install-deps 步骤自动安装。

浏览器端开发

最快速的调试方式，适合日常 UI 与逻辑开发。数据通过 Capacitor Filesystem Web 兜底实现存储到浏览器 IndexedDB。

npx vite
调试技巧
  • Chrome DevTools（F12）断点调试、查看 Network 与 IndexedDB
  • 安装 Vue DevTools 浏览器插件查看组件树与 Pinia 状态
  • Vite 自动热更新（HMR），无需手动刷新
  • 浏览器端无法调试 Electron 主进程，需在 Electron 环境中验证

Electron 端开发

通过内置 Express 服务器提供 API，数据存储在 app.getPath('userData')/data/。

pnpm build
npx electron electron/main.cjs
调试技巧
  • 主进程日志输出到终端与日志文件 userData/logs/app-YYYY-MM-DD.log
  • 渲染进程可通过 mainWindow.webContents.openDevTools() 打开 DevTools（生产已禁用）
  • 修改主进程代码后重启 Electron 生效

Android 端开发

pnpm build
npx cap sync android

然后用 Android Studio 打开 android/ 编译运行。

注意事项

  • 浏览器端与 Electron 端数据存储相互隔离，不互通
  • vite.config.ts 中 base: './' 确保打包后资源路径正确，请勿修改
  • virtual:version 虚拟模块在构建时从 package.json 注入版本号

  • Capacitor — Android
05

UI/UX 规范

视觉风格、布局与交互约定

视觉风格

配色

深色星空背景、蓝紫渐变 #667eea、琥珀强调 #fbbf24

文字

rgba 白色透明度梯度，深色背景下 clear 可读

布局规范

  • 面包屑地址栏导航，支持下拉快速切换模块
  • 卡片网格布局，响应式列数（最多 6 列）
  • 支持分屏显示（左右双面板）

交互规范

  • 卡片右上角 hover 显示操作按钮
  • 下拉菜单使用固定定位 + backdrop-filter 模糊背景
  • 确认操作使用 ConfirmDialog，消息提示使用 ElMessage

3D 场景层级

层级顺序

Canvas 流星画布的 z-index 必须低于 Three.js 元素（地球、月球、太阳），避免遮挡。

笔记幻灯片内容区

  • 幻灯片内容区去除深色背景，融入星空主题，文字浅色
  • 代码块通过"插入 HTML 代码"按钮插入，编辑/预览/放映态均语法高亮
  • 原生 pre 块自动语法高亮与格式化缩进，使用 src/utils/codeBlock.ts 的 highlightCodeBlocks / serializeCodeBlocks，编辑态保存前需调用 serializeCodeBlocks 还原纯文本
  • 主工具栏在未选中文本时禁用字体、字号、加粗等按钮，通过 hasSelection 控制
  • 下划线按钮为单按钮（点击展开下拉、不直接应用），提供 8 种线条样式与颜色设置；颜色选择器通过 Teleport 浮动到 body
  • 复制样式/应用样式按钮在主工具栏，样式保存至 data/用户ID/notes/style.json；仅当前编辑会话有效，离开时清空
  • 粘贴内容提供"保留源格式"和"只保留文本"两个选项
  • 致谢页通过右键导航栏的致谢页项，在幻灯片菜单中选择"修改致谢内容"弹窗编辑

构建与部署规范

各平台构建命令与部署流程

Windows

pnpm electron:build:win

清理 → vite build → 安装依赖 → 生成图标 → electron-builder → 修复 latest.yml

Android

通过 Android Studio 打开 android/ 目录，执行 Gradle 构建。

应用图标构建

图标由源文件 build/app-icon.png 通过构建脚本自动生成各平台所需尺寸。

build/app-icon.png（源文件）
  │
  ├─► scripts/generate-icons.js（sharp）
  │   ├─► build/icon.png(512) ~ icon-16.png   → 桌面端
  │   └─► android mipmap（5密度×3变体）       → Android
  │
  ├─► scripts/build-tools.cjs make-ico
  │   └─► build/icon.ico                      → Windows 安装包

更换图标：替换 build/app-icon.png 后重新构建即可。

发布

pnpm publish-release

发布到 GitHub generic provider，通过 electron-updater 实现自动更新。

数据管理

存储架构与关键数据路径

存储架构

Electron

Express 服务器 + 本地文件系统（YAML 后端）

Capacitor

本地 localStorage 存储

隔离

数据按用户 ID 隔离：data/<用户ID>/<类型>/<键>

关键数据路径

数据路径
窗口分辨率data/<用户ID>/settings/settings.json
清单收藏与快捷访问data/<用户ID>/list/
笔记数据data/<用户ID>/notes/notes.json
笔记分类data/<用户ID>/notes/categories.json
笔记样式data/<用户ID>/notes/style.json（仅当前会话有效）
缓存机制

storageService 使用内存缓存（Map）减少重复请求，clearCache() 可清除。
