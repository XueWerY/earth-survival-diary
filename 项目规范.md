### 2.1 新增组件时的目录选择

- 属于某个**功能模块**（足迹/专注/清单/倒数日/课程表/统计/我的）→ 放入对应 `components/<模块>/`
- 属于**跨模块通用组件** → 按类型放入 `components/common/overlay/` 或 `components/common/picker/`
- 属于**认证流程** → 放入 `components/auth/`
- 默认遵循就近原则，仅单模块使用的组件不放 common

## 三、数据持久化规则

### 3.1 存储模式

项目支持两种数据存储模式，由运行环境自动判断：

| 模式           | 环境              | 存储方式                                                                                       |
| ------------ | --------------- | ------------------------------------------------------------------------------------------ |
| Electron 模式  | electronAPI 存在  | 通过 Express 后端 API 读写 `AppData\Roaming\earth-survival-diary\data` 下的 JSON 文件                |
| Capacitor 模式 | electronAPI 不存在 | 通过 `fileStore.ts` 使用 Capacitor Filesystem 插件读写设备本地 JSON 文件，路径结构与 Electron 的 `data/` 目录完全一致 |

判断逻辑：`typeof window !== 'undefined' && !(window as any).electronAPI`

### 3.2 数据路径

Electron 模式下，所有模块数据通过 Express 后端存取，实际路径为：

| 模块    | 存储路径                                                      |
| ----- | --------------------------------------------------------- |
| 账户信息  | `data\users\<邮箱>.json`                                    |
| 足迹模块  | `data\<用户ID>\footprint\footprint.json`                    |
| 专注模块  | `data\<用户ID>\focus\favorites.json`、`records.json`         |
| 清单模块  | `data\<用户ID>\list\lists.json`、`tasks.json`                |
| 倒数日模块 | `data\<用户ID>\countdown\categories.json`、`countdowns.json` |
| 课程表模块 | `data\<用户ID>\course\courses.json`、`recorded-courses.json` |
| 我的模块  | `data\<用户ID>\profile\`                                    |
| 会话数据  | `data\<用户ID>\session\`                                    |
| 设置数据  | `data\<用户ID>\settings\`                                   |
| 系统状态  | `data\<用户ID>\system\`                                     |

### 3.2.1 足迹记录字段

- 足迹记录：活动名称、日期、起止时间、备注、心情。记录卡片支持双击快捷编辑：双击标题修改名称、时间行分为"开始时间 - 结束时间"点击弹出滚轮时分选择器（时间按钮通过 `:deep(.time-btn)` 覆写为纯文本无背景框样式，时间文字颜色使用 `var(--chalk-white-85)` 提升可读性，分隔符和时长标签颜色使用 `var(--chalk-white-60)`）、双击备注区域编辑备注（无备注时显示"双击添加备注"）。日记卡片（起止时间都为空）显示创建时间（文字颜色 `var(--chalk-white-60)`），同样支持双击名称和备注进行快捷编辑。记录卡片和日记卡片每行右侧保留圆形删除按钮（`el-button :icon="Delete" circle size="small" class="task-card-btn"`）。删除确认弹窗宽度 300px。今日总结文本所在的 `.diary-section` 设有 `padding-bottom: 16px`，与界面下边界保持固定间距。

### 3.3 数据服务层

- **`api.ts`**：认证和业务数据的 HTTP API 调用，自动适配双平台（Electron 下走 `/api` Express 后端，Capacitor 下走 `fileStore.ts` 虚拟文件系统）
- **`storageService.ts`**：模块数据的统一读写层，内置 `Map` 缓存和请求去重（`pendingRequests`），通过 API 与后端通信
- **`fileStore.ts`**：Capacitor 模式下的 TypeScript 本地文件存储实现，使用 Capacitor Filesystem 插件（`@capacitor/filesystem`）在设备本地读写 JSON 文件，路径结构与 Express 后端完全一致（如 `data/users/<email>.json`、`data/<userId>/profile/profile.json` 等）

### 3.4 数据加载规则

- 程序启动后，`App.vue` 在 `onMounted` 中调用 `initializeData()`，统一加载所有模块数据
- 各 store 提供 `loadData()` / `loadTasks()` 方法，内置 `isLoaded` 标记防止重复加载
- 各功能页面切换时不重复加载数据
- 提供 `handleRefreshData()` 手动刷新机制，通过 `clearCache()` + 重新 `initializeData()` 实现

## 四、导航栏修改规则

### 4.1 全局页面导航栏（主导航栏）

页面顶部（桌面端）或底部（安卓端）显示七个功能模块的导航栏，由 `App.vue` 统一管理。导航栏宽度 500px，水平居中，与上方和下方内容区保持一致的 16px 间距，圆角 10px。每个导航项包含图标和文字，图标在上、描述文字在下纵向排列，当前所在模块以白色加粗文字标识（`.nav-item.active` 背景为透明、无高亮框），点击可快速切换模块。导航项不使用 `title` 属性，鼠标悬停时不显示系统提示框。

**架构说明**：
- 导航栏模板位于 `App.vue`，桌面端渲染在页面顶部，安卓端渲染在页面底部（通过 `isElectron` 计算属性判断平台）
- 七个导航项依次为：👣足迹、🧘专注、📋清单、⏳倒数日、📖课程表、📊统计、👤我的
- 导航项使用按钮元素，`.nav-item` 类，选中项添加 `.active` 类（白色文字 + 加粗，无背景高亮框）
- `.nav-items-scroll` 容器支持水平滚动，滚动条隐藏，选中项自动滚动到容器居中位置
- 模块定义在 `src/composables/usePageNav.ts` 的 `MODULES`、`MODULE_ICONS`、`MODULE_LABELS`、`MODULE_ROUTES` 常量中
- 专注全屏时通过 `.nav-hidden` 类隐藏导航栏

### 4.2 面包屑地址栏（仅清单和倒数日模块）

**只有清单（mission）和倒数日（countdown）页面使用面包屑地址栏**，各自独立渲染在页面内容区顶部。其余模块（足迹、专注、课程表、统计、我的）不使用面包屑地址栏。

导航状态通过 `src/composables/usePageNav.ts` 共享 composable 管理：
- `usePageNav.ts` 提供模块级单例响应式状态（`navPath`、`navContext`），各子页面通过 `usePageNav()` 获取引用
- `App.vue` 读取 `navPath` 控制路由跳转，**不再渲染面包屑和子导航栏**
- `App.vue` 通过 `watch(route.name)` 监听路由变化，当路由切换到不同模块时通过 `restoreModuleNavPath(newName)` 恢复该模块的历史导航路径，确保清单/倒数日等模块切换回来时能还原之前打开的深层视图。应用启动时通过 `await router.replace()` 同步等待路由恢复完成后再设置 `dataInitialized = true`，避免页面在路由就绪前提前渲染导致导航状态异常
- **重要**：`watch(route.name)` 的回调是异步的，必须在所有 `await` 调用之前同步设置 `navPath`（即先调用 `pageNav.setNavPath([newName])`），然后再执行 `await setSystemStateField('currentPage', newName)` 等持久化操作。否则 `await` 期间 Vue 会完成模板重新渲染，新页面组件将读取到旧的 `navPath` 值，导致面包屑和卡片列表无法正确显示
- 各模块组件在 `onMounted` 时应**同步立即**将 `navPath` 设置为对应的根路径（如 `['mission']`），确保视图瞬间渲染，再通过异步 `initMissionState` 等函数恢复深层路径，避免出现空白页或"请选择"的中间状态
- `initMissionState` 恢复深层路径前应先检查当前 `navPath` 是否已经包含深层路径（`length > 1`），若已有多级路径说明已由外部恢复，应跳过重复恢复，避免覆盖
- **清单页面和倒数日页面各自独立渲染面包屑地址栏**，互不干扰。各页面内部计算面包屑分段、管理下拉列表状态，不再通过 `pageNav.setNavContext()` 写入共享的 `navContext` 来渲染面包屑

### 4.3 内联农历日期选择器标题行（足迹页面）

足迹页面不再使用子导航栏。跳转日期、添加足迹、添加日记三个操作按钮集成到 `InlineLunarDatePicker` 的标题行上。标题行分为左右两个区域：左侧显示年月标题文本（左对齐），右侧显示三个操作按钮（右对齐）。按钮采用"图标 SVG + 文字描述"格式（`.icon-btn` 内包含 SVG 图标和 `<span>` 文字），按钮文本大小为 12px，SVG 图标大小为 14px，按钮之间间距 4px。按钮不使用 `title` 属性，鼠标悬停时不显示描述。通过 `showHeaderActions` prop（默认 `true`）控制按钮显隐。日期选择器与下方卡片展示区之间间距 16px（`.header-actions` 的 `padding-bottom: 0`，`.diary-section` 的 `margin-top: 16px`，使 `.header-actions` 底部的横线到日期标题之间的视觉间距为 16px）。`.diary-section` 设有 `padding-bottom: 16px`，确保今日总结文本与界面下边界保持固定间距。

### 4.4 专注页面控制按钮

专注页面不再使用子导航栏。切换番茄钟（🍅 番茄钟）和切换正计时（⏱️ 正计时）两个切换按钮采用"emoji + 文字"格式，位于星轨时钟下方 `.focus-control-buttons` 容器内。开始专注（▶️ 开始专注）按钮独立放在两个输入框（专注事项、备注）下方，通过 `.focus-start-wrapper` 容器水平居中，样式与切换按钮一致（透明背景、白色半透明边框、无渐变色）。所有按钮不使用 `title` 属性，鼠标悬停时不显示描述。当前模式按钮（番茄钟或正计时）高亮显示。

计时运行中，取消和完成按钮（✕ 取消、✓ 完成）显示在计时器下方，采用与切换按钮相同的"图标 + 文字"样式。取消按钮悬停时显示红色主题，完成按钮默认带蓝色边框且文字为白色，悬停时蓝色加深。

数据层级为：**模块 > 文件夹 > 清单 > 分组 > 任务**（清单模块）/ **模块 > 分类**（倒数日模块）。

1. **路径模型**：使用 `navPath: string[]` 表示当前导航路径，第一个元素为模块名。仅清单和倒数日模块使用多级路径，其余模块仅根路径一个元素：
   - `['footprint']` / `['focus']` / `['course']` / `['statistics']` / `['profile']` = 对应模块页面（无面包屑）
   - `['mission']` = 清单模块首页（展示智能清单文件夹卡片 + 自定义文件夹卡片）
   - `['mission', 'smart']` = 智能清单概览
   - `['mission', 'smart', type]` = 智能清单任务详情（type = today/expired/future）
   - `['mission', 'custom']` = 自定义文件夹概览
   - `['mission', 'custom', folderId]` = 文件夹内清单列表
   - `['mission', 'custom', folderId, listId]` = 清单内分组列表
   - `['mission', 'custom', folderId, listId, groupId]` = 分组内任务列表
   - `['countdown']` = 倒数日模块首页（展示分类卡片，含"全部"分类卡片优先显示）
   - `['countdown', categoryId]` = 倒数日分类详情（`categoryId === 'all'` 时显示所有倒数日）

2. **面包屑渲染**（仅清单和倒数日）：各页面自主渲染面包屑地址栏（`.mission-breadcrumb-bar` / `.countdown-breadcrumb-bar`），格式为 `模块图标 > 文件夹名 > 清单名 > 分组名`（清单）或 `模块图标 > 分类名`（倒数日）。倒数日分类面包屑的下拉列表第一项为"全部"虚拟分类（紫色显示）。模块图标采用按钮元素点击回到模块首页。所有文本段（除 `>` 和模块图标外）均使用对应颜色渲染。文本过长时 `.breadcrumb-scroll` 支持横向滚动，滚动条隐藏；navPath 变化时自动滚动到最右侧。

3. **交互规则**（面包屑，仅清单和倒数日）：点击模块图标回到对应模块首页；面包屑中有 `dropdownItems` 的分段前面会显示可点击的 `>`，点击弹出对应层级的子项列表，选中项高亮，点击后导航到对应视图，点击主内容区域关闭下拉列表。

4. **导航栏样式**：主导航栏 `.main-nav-bar` 宽度 500px 水平居中（`margin: 16px auto`），圆角 10px，背景半透明 `rgba(255, 255, 255, 0.03)`，底部导航栏（安卓）有上边框分隔。`.nav-items-scroll` 内导航项水平排列，图标在上、文字在下纵向排列，选中项以白色加粗文字标识（无背景高亮框）。清单页面面包屑地址栏 `.mission-breadcrumb-bar` 和倒数日页面面包屑地址栏 `.countdown-breadcrumb-bar` 宽度为页面宽度的 80%，水平居中，与顶部导航栏之间通过 `margin-top: 8px` 保持间距，与下方内容区通过 `margin-bottom: 16px` 保持间距，圆角 8px。各页面面包屑地址栏背景半透明，内部 flex 布局。模块图标按钮和 Plus 添加按钮无背景框，hover 显示淡紫色背景。专注全屏时 `.nav-hidden` 类隐藏导航栏。段落下拉列表定位在面包屑栏下方，左边界与对应的箭头左边界对齐（通过 `getBoundingClientRect` 动态计算）。

5. **页面操作按钮**：
   - **足迹页面**：跳转日期、添加足迹、添加日记按钮位于日期选择器标题行右侧，按钮采用"SVG 图标 + 文字"格式，不使用 `title` 属性。记录卡片和日记卡片使用 `TimePickerPopover` 组件实现行内时间选择，双击标题和备注区域触发多行文本框（`textarea.inline-edit-textarea`）进行快捷编辑，无需弹窗。
   - **专注页面**：切换番茄钟和切换正计时按钮采用"emoji + 文字"格式位于星轨时钟下方；开始专注按钮独立放在输入框下方，水平居中，样式与切换按钮一致（无渐变色背景）；计时中的取消和完成按钮采用同样的"图标 + 文字"样式。所有按钮不使用 `title` 属性
   - **清单模块首页**：添加文件夹（+）
   - **自定义文件夹概览/文件夹内清单列表**：添加清单（+）
   - **分组列表**：添加分组（+）
   - **分组内任务列表**：添加任务（+）
   - **倒数日模块首页**：添加分类（+）
   - **倒数日模块分类列表**：添加倒数日（+）
   - **课程表/统计/我的**：无页面操作按钮

5b. **倒数日卡片交互规则**：
   - 卡片按钮（星标、删除）默认隐藏（`opacity: 0`），鼠标悬浮卡片时显示（`opacity: 1`），按钮无背景框无边框无阴影
   - 删除现有编辑按钮（`Edit` 图标），编辑操作改为双击卡片名称区域触发多行文本框（`textarea.inline-edit-textarea`）进行编辑，失去焦点或按 Enter 保存，按 Escape 取消
   - 日期显示为纯文本（`.date-plain-text`），无背景框无边框，点击弹出 `DateScrollPicker` 滚轮式农历日期选择器直接修改
   - 重复策略显示为可点击文本：仅 `countMode !== 'countup'` 时显示该区域，`repeatStrategy === 'yearly'` 时显示"每年重复"（蓝色），否则显示"不重复"（灰色），点击直接切换。正数日（countup）卡片保留 `milestone-meta` 容器占位（通过 `min-height: 18px` 维持高度），内部不显示重复策略和提醒设置内容，确保与倒数日卡片行对齐
   - 提醒设置在卡片上显示为可点击文本：已设置提醒时显示提醒标签（橙色），未设置时显示"不提醒"（灰色），点击弹出提醒设置弹窗（350px 宽，标题居中无关闭图标，底部图标胶囊按钮）
   - 描述区域：有描述时显示描述文本，无描述时显示"双击添加描述"灰色斜体提示文本，双击触发多行文本框编辑
   - **提醒调度适配**：`App.vue` 的提醒调度模块在处理倒数日提醒时，若里程碑的 `repeatStrategy === 'yearly'`，会调用 `getNextOccurrence(targetDate, 'yearly')` 将目标日期滚动至下一个未过期的年份对应日期，再基于滚动后的日期计算提醒触发时间。确保"每年重复"的倒数日在过去后次年仍能正常提醒。
   - **系统里程碑**：所有系统自动创建的倒数日里程碑（节日、生日、使用天数等）默认设置 `repeatStrategy: 'yearly'`，正数日里程碑同样设置为每年重复。
   - **提醒重新调度**：当用户通过卡片修改日期（DateScrollPicker）、切换重复策略（点击文本切换）、或通过弹窗修改提醒设置后，自动调用 `refreshReminders()` 触发提醒重新调度。
   - **页面样式**：倒数日页面 `.content-wrapper` 隐藏滚动条（`scrollbar-width: none; -ms-overflow-style: none; 及 ::-webkit-scrollbar { display: none }`），同时通过 `:deep(.el-scrollbar__bar) { display: none !important; }` 隐藏 Element Plus 滚动条组件的水平和竖直滑块，保留滚动功能；`.section` 统一使用 `padding: 20px 0; background: transparent`，无背景框无水平内边距，卡片占满 80% 内容区宽度。

6. **导航状态管理接口**（`usePageNav.ts`）：
   - `navPath`: `Ref<string[]>` — 当前导航路径
   - `navContext`: `Ref<NavContext>` — 面包屑数据 + 操作按钮配置
   - `setNavPath(path: string[])` — 更新导航路径
   - `setNavContext(ctx: NavContext)` — 更新面包屑上下文
   - `goModuleHome()` — 回到当前模块首页
   - `restoreModuleNavPath(module: string): Promise<string[]>` — 从持久化存储恢复模块的导航路径。严格验证恢复的路径第一个元素必须等于 `module`，不匹配时放弃恢复并返回默认根路径 `[module]`，防止错误的跨模块路径覆盖当前页面

7. **NavContext 接口**：
   - `segments: BreadcrumbSegment[]` — 面包屑分段列表（模块图标右侧的文本段）
   - `plusVisible: boolean` — 是否显示右侧 + 按钮（当 `actions` 存在时忽略此字段）
   - `plusOnClick: (() => void) | null` — + 按钮点击处理函数
   - `goModuleHome: () => void` — 回到模块首页的回调
   - `actions?: NavAction[]` — 自定义操作按钮列表（可选，优先于 plusVisible/plusOnClick 渲染）

7b. **NavAction 接口**：
   - `icon: string` — 按钮显示的图标文本（emoji 字符，如 📅、➕、📝、🍅、⏱️、▶️）
   - `onClick: () => void` — 按钮点击处理函数
   - `title: string` — 鼠标悬停提示文本

8. **BreadcrumbSegment 接口**：
   - `label: string` — 显示文本
   - `color: string` — 文本颜色
   - `clickable: boolean` — 是否可点击（对应父级面包屑）
   - `onClick: (() => void) | null` — 点击回调
   - `dropdownItems: DropdownItem[] | null` — 下拉列表项（非 null 时前面 `>` 可点击弹出下拉）

9. **DropdownItem 接口**：
   - `id: string` — 唯一标识
   - `name: string` — 显示名称
   - `color: string` — 圆点颜色
   - `current: boolean` — 是否为当前项
   - `onSelect: () => void` — 选中回调

10. **状态持久化**：清单模块 navPath 通过 `setSystemStateField('list', { navPath })` 持久化存储；倒数日模块 navPath 通过 `setSystemStateField('countdown', { navPath })` 持久化存储。持久化由 `usePageNav` 内部的 `watch(navPath)` 自动触发。**仅持久化深层路径（length > 1）**，根级路径（如 `['mission']`）不持久化，避免覆盖已保存的深层路径。恢复时 `restoreModuleNavPath` 严格验证路径首元素必须匹配当前模块名，不匹配则返回 `[module]` 默认根路径，防止跨模块路径污染导致页面空白。旧版导航数据（不含模块前缀）自动迁移为新版格式，无需手动操作。

11. **子页面接入方式**：子页面通过 `usePageNav()` 获取 `pageNav` 实例，使用 `pageNav.navPath`（computed）读取路径、`pageNav.setNavPath()` 更新路径。清单和倒数日页面通过 `watch` 监听 navPath 和相关数据变化，在本地计算面包屑分段并渲染，同时管理自己的下拉列表状态，**不再调用 `pageNav.setNavContext()`**。

## 六、状态管理规范

### 6.1 Store 设计模式

所有 Pinia Store 使用 **Setup Store（Composition API）** 风格。统一遵循以下模式：

```typescript
export const useXxxStore = defineStore('xxx', () => {
  const data = ref<Xxx[]>([])
  const isLoaded = ref(false)

  const loadData = async () => {
    if (isLoaded.value) return  // 防止重复加载
    // ... 加载逻辑
    isLoaded.value = true
  }

  const reset = () => {
    data.value = []
    isLoaded.value = false
  }

  return { data, isLoaded, loadData, /* ... */, reset }
})
```

**命名约定**：

- 获取数据：`loadXxx()` / `getXxx()`
- 添加：`addXxx()`
- 更新：`updateXxx()`
- 删除：`deleteXxx()`
- 切换状态：`toggleXxx()`
- 重置：`reset()`（用于登出时清理）

### 6.2 Store 职责划分

| Store           | 职责                  | 数据来源             |
| --------------- | ------------------- | ---------------- |
| `authStore`     | 用户认证、个人信息、登录/注册/登出  | `api.ts`         |
| `taskStore`     | 足迹记录增删改查            | `api.ts`         |
| `focusStore`    | 专注记录、常用专注、计时状态、统计计算 | `storageService` |
| `missionStore`  | 清单/分组/任务管理、提醒策略     | `api.ts`         |
| `settingsStore` | 专注设置、课程表设置、提醒设置     | `api.ts`         |

## 七、编码规范

### 7.1 写前先读

在修改任何文件之前，必须先用 Read 工具完整阅读该文件，理解其现有逻辑、代码风格和依赖关系。禁止在不了解文件内容的情况下直接修改。

### 7.2 日志记录

- 统一使用 `logger.ts`（`import { logger } from 'path/to/lib/logger'`）
- 支持 5 个级别：`trace`、`debug`、`info`、`warn`、`error`
- 日志自动上传到服务端（`/api/logs`），同时输出到浏览器控制台
- 日志格式：`[时间戳] LEVEL: 消息 + meta 对象`
- 关键操作（路由切换、数据操作、登出、错误）必须添加日志
- **调试日志覆盖**：清单页面（`MissionList.vue`）、导航状态管理（`usePageNav.ts`）、清单数据存储（`missionStore.ts`）、倒数日页面（`CountdownList.vue`）、足迹页面（`TaskList.vue`）、应用入口（`App.vue`）均已添加完整的 `logger.debug` 日志，覆盖以下关键节点：
  - `setNavPath` 调用（含调用栈、前后值）
  - `restoreModuleNavPath` 完整流程（读取存储、路径匹配验证、回退默认值）
  - `navPath` watch 持久化触发
  - MissionList 组件挂载（guard 检查、loadData、initMissionState）
  - showBreadcrumb / isMissionRoot 计算属性重算
  - breadcrumb segments watch 触发
  - App.vue 路由恢复（router.replace 前后状态、dataInitialized 设置）
  - App.vue route watch（新旧路由名、currentModule、navPath）
  - missionStore.loadData（跳过/开始/获取/完成/失败）
  - CountdownList / TaskList onMounted（navPath 状态、初始化步骤）

### 7.3 删除二次确认

任何涉及删除操作的代码，必须添加二次确认弹窗，防止用户误操作。统一使用 `ConfirmDialog.vue` 组件。

### 7.4 数据统一加载

程序启动后统一加载所有功能页面的数据（在 App.vue 的 `initializeData` 中集中处理），各功能页面切换时不再重复加载数据。如需刷新，提供手动刷新机制。

### 7.5 TypeScript 严格模式

- `strict: true`
- `noUnusedLocals: true` — 不允许未使用的局部变量
- `noUnusedParameters: true` — 不允许未使用的参数
- `erasableSyntaxOnly: true` — 仅允许可擦除的类型语法
- 所有组件使用 `<script setup lang="ts">`

### 7.6 `<script setup>` 变量声明顺序（避免 TDZ 错误）

在 `<script setup lang="ts">` 中，所有 `const`/`let` 声明按代码顺序执行且存在临时死区（Temporal Dead Zone），引用尚未初始化的变量会抛出 `ReferenceError: Cannot access 'xxx' before initialization`。必须遵守以下顺序规则：

- **`watch` 和 `watchEffect`（含 `immediate: true`）必须放在所有被引用的 `ref`、`computed`、`reactive` 和函数声明之后**。因为带有 `immediate` 的 watch 在创建时立即执行回调，如果回调或依赖数组中引用了后续声明的变量，会触发 TDZ 错误
- 即使是不带 `immediate` 的 watch，其依赖数组在创建时也会被求值，因此同样不能引用后续声明的变量
- 计算属性（`computed`）可以安全引用后续声明的函数（因为计算属性是惰性求值的，首次求值在模板渲染时，此时所有变量已初始化），但不能引用后续声明的 `ref`/`computed`/`reactive`（因为计算属性的 getter 可能在依赖收集时立即被调用）
- **结构规范**：建议所有 `ref` 和 `computed` 声明集中在 setup 前半部分，所有 `watch` 声明放在所有数据声明之后

### 8.1 编码先思考

- 用最少的代码解决问题，不编写任何多余臆测内容。
- 不开发需求以外的额外功能。
- 仅单次使用的代码，不做抽象封装。

- 不对不可能出现的场景编写异常处理逻辑。

- 只改动必要内容，仅清理自身修改产生的冗余。
- 不擅自优化周边代码、注释、代码格式。
- 不对运行正常的代码进行重构。
- 遵循原有代码风格（缩进、命名、导入方式等），即便有更好的编写方式。
- 若发现无关的无用代码，仅备注说明，不擅自删除。
- 删除本次修改后不再使用的导入语句、变量、函数。但未经要求，不删除原本就存在的无用代码。

## 十三、提醒系统

### 13.1 架构

提醒由 Electron 主进程管理，前端提交调度，主进程负责定时触发：

1. 前端收集所有需要提醒的任务/倒数日事件
2. 调用 `window.electronAPI.scheduleReminders(reminders, persistDuration)` 提交调度
3. 主进程维护 `reminderTimers` 数组和 `reminderQueue` 队列
4. 触发时通过 `show-reminder` IPC 事件发送到渲染进程
5. 渲染进程右下角面板显示提醒，播放提示音

### 13.2 提醒数据格式

```typescript
interface ReminderItem {
  id: string
  name: string           // 提醒名称
  body: string           // 提醒内容
  triggerTime: string    // 触发时间 ISO
  listName?: string      // 所属清单名
  groupName?: string     // 所属分组名
  repeatStrategy?: string // 重复策略
  // ...
}
```

### 13.3 提醒重复调度

- 支持 `daily` / `weekly` / `monthly` / `yearly` / `hourly` / `custom_days` / `weekdays`
- 最大调度时间窗口 20 天（`MAX_SCHEDULE_DELAY`），超过的等下次启动重新调度
- 登出时调用 `cancelAllReminders()` 取消所有提醒
